1. 깃 소개
    1) 2005년 리누스 토르발스는 깃을 세상에 소개함
    2) 소스 코드의 관리 효율성을 위한 SW
    3) 핵심 기능
        - 버전 관리 (Version Control) 시스템 
        - 백업 (backup)
            -   github 가 가장 많이 사용되고 있음
                - 온라인 저장소 or 원격 저장소   
        - 협업 (collaboration)

2. 깃 환경 설정하기
    1) 깃을 사용하기 전에 먼저 사용자 정보를 입력해야 함
    2) 깃은 버전을 저장할 때마다 그 버전을 만든 사용자 정보도
        함께 저장함
        
    3) 사용자 이름 설정 
        git config --global user.name [사용자 이름]

    4) 사용자 메일 설정
        git config --global user.email [사용자 메일]

3. 리눅스 명령
    1) 현재 디렉토리 확인하기
        - 깃 배시 실행 후 맨 끝에 물결 표시 (~)가 있음
            -   현 위치가 홈 디렉토리 (home directory)라는 의미
        - pwd   (print working directory)
            -   현재 위치의 경로 출력

    2) ls (list)
        -   현재 디렉토리에 어떤 파일이나 디렉토리가 있는 지 확인할 때 사용
        - -l 옵션
            - 파일이나 디렉토리의 상세 정보까지 표시함

    3) clear 
        -   터미널 창 지우기

    4) ..
        -   상위 디렉토리
    
    5) .
        -   현재 디렉토리

    6) ~
        -   홈 디렉토리

    7) 디렉토리 삭제
        -   rm -r   : 하위 폴더까지 내용 삭제
